,Composition

Distributions:
DISCRETE  (counts, integer outcomes)
├── Yes/no trial(s)
│   ├── single trial               →  Bernoulli
│   └── n fixed trials             →  Binomial
├── Trials until success
│   ├── until 1st success          →  Geometric
│   └── until k-th success         →  Negative Binomial
└── Count of events in an interval →  Poisson


CONTINUOUS  (real-valued measurements)
├── How is the value generated?
│   ├── sum of small independent effects →  Normal
│   └── product of many small effects →  Lognormal
├── Waiting / service time
│   ├── memoryless                    →  Exponential
│   ├── with aging / wear-out         →  Weibull
│   └── multi-stage (k stages)        →  Gamma / Erlang
└── Bounded value
    ├── min / mode / max known        →  Triangular
    ├── flexible shape on [a, b]      →  Beta
    └── range only, otherwise unknown →  Uniform

Probability Mass Function PMF: probability for discrete outcome like X=10
Probability Density Function PDF: probability of an interval describes the area beneath function

For discrete tests Chi-square, we pit observed data against expected bin counts of histogramms. (alpha and beta later)
For continuous data we use KS-test to compare empirical with theoretical CDF.
Type 1 error is a false alarm; distribution fits but still reject it.
Type 2 error is a missed misfit; distribution doesnt fit but accept it.
P-value after such tests represents probability of a gap between candidate and data distributions
under the test H0 assumption. Higher p indicates higher likelihood of compatibility
If p > alpha then you fail to reject H0, otherwise you reject H0 (aka you accept H1)
For H0, we have alpha and beta. Alpha is the threshold if p < alpha. This is the type 1 error we accept.
For KS-test: you look at the vertical difference between the functions. Lower gap = better fit
